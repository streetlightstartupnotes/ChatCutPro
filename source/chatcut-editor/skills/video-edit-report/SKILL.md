# video-edit-report

生成剪辑报告：汇总所有编辑操作的结果，展示删了什么、节省多少时间、输出了哪些文件。

## 触发条件

用户要求"剪辑报告""统计""report""看看效果"时调用。通常在所有剪辑操作完成后调用。

## 输入

- `project_dir`: 项目目录路径

## 执行步骤

1. 读取 `project.json` 获取原始视频元数据

2. 扫描项目目录下的所有输出文件，用 `ffprobe` 获取各文件时长

3. 计算：
   - 原始时长 vs 各阶段输出时长
   - 删停顿节省的时间
   - 删口癖节省的时间
   - 总节省时间和比例

4. 如果有转写数据，统计：
   - 总词数
   - 口癖词频率
   - 停顿次数和总时长

5. 生成 Markdown 报告：
   ```markdown
   # 剪辑报告

   ## 基本信息
   - 原始文件：video.mp4
   - 原始时长：5:30.0
   - 最终时长：4:12.3
   - **总计节省：1:17.7（23.5%）**

   ## 操作明细
   | 操作 | 删除数量 | 节省时间 |
   |------|----------|----------|
   | 删停顿 | 23 处 | 45.2s |
   | 删口癖 | 17 处 | 32.5s |

   ## 口癖词排行
   1. 嗯 (8次)
   2. 那个 (5次)
   3. 就是 (4次)

   ## 输出文件
   - silence_cut.mp4 (42.1 MB)
   - filler_cut.mp4 (38.7 MB)
   - captions.srt
   - export_9x16.mp4 (35.2 MB)
   ```

## 依赖

- FFmpeg / ffprobe

## 输出

- Markdown 格式的剪辑报告
- 也可输出为 JSON 供 UI 展示
