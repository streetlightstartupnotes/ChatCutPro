# 总体规划

目标：把 ChatCutPro 做成“左侧可实时操作剪辑区 + 右侧自然语言剪辑 Agent”的 ChatCut 复刻产品。用户可以上传视频，用聊天下指令，也可以像剪映一样直接操作时间线、字幕、片段和导出。

## 产品原则

- 第一屏就是可用剪辑器，不做营销页。
- 左侧承载所有实时剪辑操作，右侧承载 Agent 对话和复杂意图。
- 小白优先：每个状态都应该能看懂下一步。
- 不做快捷键，直到用户重新要求。
- 预算有限时优先补“可用闭环”：导入、看懂、剪掉、改字幕、预览、导出。

## 阶段 1：让现有功能更可懂

- 把左侧分成明确区域：预览、文稿、时间线、属性/导出。
- 将“精剪、字幕、导出”的状态结果可视化。
- 给字幕保存、渲染、导出加明确状态和失败提示。
- 统一术语：面向用户用“修改记录、片段、字幕、音乐、旁白”。

## 阶段 2：补齐剪映式基础编辑

- 时间线片段拖拽排序、拖动边缘 trim。
- 轨道开关：显示/隐藏、静音、锁定。
- 片段属性：起止时间、音量、透明度、字幕样式。
- 文稿剪辑与时间线联动：点文字跳预览，裁掉文字即裁时间线。

## 阶段 3：复刻 ChatCut 核心智能能力

- 智能粗剪：去停顿、去口癖、去重复、保留高光。
- 智能字幕：分行、关键词高亮、模板样式、实时编辑。
- 智能包装：标题卡、转场、贴纸、MG 动效。
- 智能 B-roll：根据文稿推荐素材，并允许一键替换。
- 智能改稿：自然语言要求“更短、更有冲突、更像口播”等，生成可审阅修改方案。

## 阶段 4：专业输出和可迁移

- 导出多比例、多码率、多平台 preset。
- 导出字幕文件、封面、剪辑报告。
- 支持 XML/FCPXML/OTIO 这类专业工程交接格式。
- 项目打包和恢复，方便跨机器继续编辑。

## 阶段 5：稳定性和性能

- 长视频分段渲染与缓存。
- 渲染队列、取消、重试。
- 大项目只更新变更片段，避免每次字幕编辑都全量重渲染。
- 自动备份和版本差异可视化。
